Runtime Controls & Execution Model

Script execution, hotkeys, and the live dashboard

Execution Model
Parallel Scripts
• Multiple scripts run simultaneously
• Each script gets its own match loop thread
• Shared resource lock for input actions
• Max concurrent: 8 (configurable)
Match Loop (per script)
1. Capture target window region
2. Run all active triggers' template matching
3. Dispatch to matching trigger's action tree
4. Evaluate conditions → execute actions
5. Sleep for tick interval → repeat
Resource Limits
• CPU: auto-throttle when < 1 match/s
• Match cache: avoid re-scanning static areas
• Batch captures for multiple triggers on same window
