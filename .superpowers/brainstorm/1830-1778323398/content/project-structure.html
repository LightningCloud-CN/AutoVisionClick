Error Handling Strategy

Scenario Behavior User sees
Window not found Script pauses, retries every 2s, auto-resumes when window reappears ⚠ "Window 'Game' not found — waiting..."
Template match timeout No match for N seconds → trigger optional fallback action ⏱ "No match for accept.png in 30s"
Click blocked / app foreground lost Queue action, retry on next tick if window is foreground again ⚠ "Input blocked — window not in foreground"
Invalid script JSON Validation on load — refuse to run, highlight broken nodes in editor ✕ "Script 'X' has 2 errors — see editor"
Emergency kill (Ctrl+Shift+F8) Immediately: stop all threads, release input hooks, save state 🛑 "EMERGENCY STOP — all scripts halted"
Unhandled engine crash Per-script isolation — one script crash does not kill others ✕ "Script 'X' crashed — check log. Other scripts unaffected."

Validation Rules (editor-side)

🔴 Errors (blocks run)
  • Trigger with no template selected
  • Action with missing required field
  • Loop with no exit condition
  • Circular group references
🟡 Warnings (script runs anyway)
  • Confidence threshold < 0.7
  • Wait < 100ms (may flood input)
  • Forever loop with no delay
  • Unused templates in project
